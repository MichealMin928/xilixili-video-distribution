# 洗哩洗哩四平台日常运营流程

## 零、每天检查

1. 运行 `xilixili-service status` 确认本机服务正常；异常时运行 `xilixili doctor`。
2. 使用 `$xilixili-daily-ops`，或先执行 `xilixili-daily status` 检查当天状态。
3. 对当天内容执行 `xilixili-daily plan <manifest>`；重复内容必须更换，不用 `--force` 凑数。
4. 执行 `xilixili-daily start <manifest>`，程序自动检查登录、建立或复用任务并预填四个平台。
5. 检查预览与截图。用户批准后，在同一工作时段分别执行 `xilixili-daily schedule <任务编号> <平台>`，由四个平台原生定时发布，不等待计划时刻再点击。

当前使用 macOS 当前用户的 LaunchAgent 保持运营台运行，并使用 Codex 周期任务准备已批准内容、提交平台原生定时。若服务中途重启，当天任务应重新复核预填与定时提交状态。

## 零点二五、评论与私信由账号本人处理

不再运行定时互动巡检；内容准备、定时提交、发布或日常平台运营结束后，也不自动打开评论、评论回复或私信。四个平台的互动由账号本人查看并回复。

只有用户在当前对话中重新明确要求 Codex 检查互动时，才可执行一次只读 `xilixili-engagement scan all`；该次授权不延续到以后任务，也不授权 Codex发送任何回复。

当前使用普通抖音账号，不访问或要求抖音企业号线索版。

## 零点五、每周自动节奏

周二、周五 18:30 自动预填当天已批准内容，并在同一次任务中依次提交视频号、快手、抖音和小红书的原生定时。实际发布时刻以各平台当天允许并返回的时间为准，不再建立四个到点点击任务。详细授权边界和周期任务见 `records/每周发布排期.md`。

自动发布只处理 `content/approved/` 中且包含 `"approvedForAutoPublish": true` 的清单。普通草稿、测试素材和未明确批准的内容不会发送。

## 零点七五、每周数据清理

每周一 03:30（北京时间）运行 `npm run cleanup -- --apply`。清理范围采用白名单：操作、互动和核验截图保留 30 天，发布成功凭证保留 365 天，归档日志保留 30 天。正式稿件、运营记录、发布状态、互动状态、浏览器登录资料和当前服务日志不参与自动清理。

需要人工检查清理范围时，运行 `npm run cleanup` 进入预演模式；预演只列出候选文件，不删除任何数据。

## 一、准备内容

每条内容建立一个任务，至少包含：

- 内容类型：图文或视频
- 基础标题与正文
- 本地素材文件
- 相关话题
- 计划发布时间（如有）

账号统一定位：江西酒店投影＋江西酒店供应链资源。

内容以“投影品类和酒店使用场景”为主，不把账号绑定为单一品牌；涉及携程合作、自营标签或指定品牌等表述时，必须先核对当期证明材料，并避免把单个产品或阶段性合作扩大为长期、全品类背书。

## 二、四平台预填

1. 创建内容任务，系统自动生成小红书、抖音、快手、微信视频号版本。
2. 选择排期日期，核对各平台原生定时控件给出的有效时间；必须与工作流日期相同且距离当前时间超过 5 分钟，抖音还需满足至少提前 2 小时的限制。
3. 依次执行四个平台预填。
4. 系统上传素材、填写标题/正文/话题并停在发布按钮前。
5. 快手没有独立标题栏，标题自动放在作品描述第一行。
6. 平台话题数采用固定差异：抖音 5 个、小红书 5 个、快手 4 个、视频号 3 个；系统按基础话题顺序裁剪。
7. 基础话题按“酒店投影、江西酒店、买家意向词、问题词、辅助词”排序，确保话题较少的平台仍保留核心业务与地域。
8. 每个平台保存一张预览截图，并把结果写入本地审计记录。

### 抖音图文音乐标准步骤

1. 抖音图文默认添加与内容气质匹配的轻音乐、钢琴或氛围纯音乐，避免图片只滚动而显得单调。
2. 优先无人声、不抢标题与图片信息的舒缓配乐；不选伤感情歌、强节奏或与酒店投影无关的热门音乐。
3. 提交前必须确认音乐名称已经回填或页面明确显示已添加音乐；无法确认时停在发布前并提示人工检查。

### 快手图文音乐标准步骤

1. 完成图片、作品描述和最多 4 个话题的预填。
2. 点击“添加音乐”，在页面右侧打开“选择音乐”抽屉。
3. 搜索“轻音乐”，优先选无人声、不抢文案信息的纯音乐；在右侧曲目行悬停并选择。
4. 抽屉关闭后，核对音乐名称已回填，右侧作品检测显示“作品已添加音乐”。
5. 选择原生定时时间并提交，只有跳转到作品管理页或出现“审核中/发布成功”才记为成功。
6. 若发布按钮不可用，先检查话题数是否超过 4 个，再检查音乐是否已回填；不连续盲点发布。

## 三、发布前人工确认

确认以下内容后才可发布：

- 画面、字幕、封面没有错字或裁切
- 抖音图文和快手图文已添加合适的纯音乐，且页面能确认选中状态
- 产品参数、价格、合作关系和地域范围有依据
- 文案没有绝对化承诺、虚假背书或无法证明的“官方/指定/第一”等表述
- 联系方式、门店信息和服务城市准确
- 四个平台预览与各自界面规则相符

## 四、正式发布

手工正式发布必须输入与任务编号完全相同的二次确认值。周期任务只接受 `content/approved/` 目录与 `approvedForAutoPublish=true` 的双重站立授权。批准后优先逐个平台提交原生定时，记录平台选中的实际时刻；不建立到点唤醒任务，也不在线等待。只有用户明确要求立即发布时才使用 `xilixili-daily publish`。

第一条正式内容需逐平台观察发布结果；验证通过后，后续同类内容可继续使用统一流程。

## 五、发布后记录

记录每个平台的发布时间、作品链接、成功/失败状态和必要备注。出现验证码、风控提示、页面改版或素材审核问题时停止该平台操作，保留其他平台任务和截图，不反复点击。
